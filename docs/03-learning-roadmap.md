# 学习路线图

## 阶段 1：先跑通，不求全懂

目标：

- 能启动 MySQL、ZooKeeper、provider、consumer
- 能成功调通 `/user/list`

你现在最需要建立的是“信心”和“整体感”，不是细抠每个注解。

建议关注：

- 根 `pom.xml` 里有哪些依赖版本
- provider 和 consumer 的配置差异
- HTTP 请求是怎么转成 Dubbo 调用的

## 阶段 2：按调用链读代码

建议从外到内读：

1. `UserController`
2. `UserService` 接口
3. `UserServiceImpl`
4. `UserMapper`
5. `User` / `UserDTO` / `Result`

每读完一层，都问自己 3 个问题：

1. 这一层的职责是什么
2. 它依赖谁
3. 它为什么不应该做别人的事

## 阶段 3：理解框架是怎么接起来的

这个阶段重点不是业务，而是框架协作关系：

- Spring Boot 负责应用启动和 Bean 管理
- Dubbo 负责远程服务调用
- ZooKeeper 负责服务注册与发现
- MyBatis-Plus 负责数据库访问

如果能把这四句话讲明白，你就已经入门了。

## 阶段 4：带着目标做小改造

建议按从简单到复杂的顺序改：

1. 给 `User` 增加一个 `age` 字段
2. 给新增接口加非空校验
3. 给查询接口加分页
4. 给项目加统一异常处理
5. 给 consumer 返回更友好的错误信息

为什么这样安排：

- 改字段：练数据库、实体、DTO、接口联动
- 加校验：练 Spring Web 常见能力
- 加分页：练 MyBatis-Plus 常用功能
- 加异常：练工程化思维

## 阶段 5：开始像真实项目一样思考

当你完成前面的内容后，可以继续问这些问题：

- DTO 和 Entity 应该完全复用吗
- Consumer 是否应该直接返回 Provider 的 `Result`
- Dubbo 超时和重试该怎么配
- Provider 的业务逻辑是否应该再拆 service 层
- 如何区分开发、测试、生产环境配置

## 推荐你的学习节奏

如果你每天学 1 到 2 小时，可以这样安排：

1. 第 1 天：跑通项目，理解模块结构
2. 第 2 天：读完用户查询和新增链路
3. 第 3 天：自己手改一个字段并跑通
4. 第 4 天：加参数校验和异常处理
5. 第 5 天：加分页并整理笔记

## 学完这个项目后，你应该能掌握什么

至少能掌握这些基础能力：

- 看懂 Maven 多模块 Spring Boot 项目
- 理解 Dubbo 服务提供者和消费者的基本关系
- 能独立配置 ZooKeeper 注册中心
- 能用 MyBatis-Plus 完成基础 CRUD
- 能自己扩展一个简单业务接口

这时你再去学更复杂的微服务项目，会轻松很多。
